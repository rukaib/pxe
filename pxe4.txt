## 🪟 **When Windows Server is BETTER**

### ✅ Use Windows Server If:

#### 1. **You're Already in Microsoft Ecosystem**
```
If you already have:
- Active Directory domain
- Windows infrastructure
- WSUS for updates
- System Center tools
- Microsoft licensing

→ WDS integrates seamlessly
→ Single pane of glass management
→ Native AD integration
```

#### 2. **You Need Enterprise Features**
```
Windows Server WDS offers:
✓ MDT integration (Microsoft Deployment Toolkit)
✓ Unattended installation templates
✓ Driver injection per model
✓ Application deployment during install
✓ Computer naming automation
✓ Domain joining during deployment
✓ Official Microsoft support
```

#### 3. **You Deploy Windows ONLY**
```
If you ONLY deploy Windows (no Linux/other OS):
- WDS is purpose-built for this
- Zero configuration for Windows
- Better Windows compatibility
- Automatic driver handling
```

#### 4. **You're Not Comfortable with Command Line**
```
Windows Server = GUI all the way
- Point and click
- Wizards for everything
- Less intimidating for beginners
- Visual feedback
```

## 🛠️ **Practical Setup - Linux (Recommended)**

Here's the EXACT setup I recommend for you:

### **Hardware Requirements**

```
Minimum (Will Work):

- Disk: 20GB (for OS + one Windows ISO)
- NIC: Gigabit Ethernet

Recommended (Smooth Experience):
- CPU: Core i3 or equivalent
- RAM: 2GB
- Disk: 100GB (multiple Windows versions)
- NIC: Gigabit Ethernet
```

### **Step-by-Step Linux Setup**

```bash
# ============================================
# PROVEN WORKING CONFIGURATION
# Tested on Ubuntu Server 22.04 LTS
# ============================================

# 1. Install Ubuntu Server
# Download: https://ubuntu.com/download/server
# Installation: Select "Ubuntu Server (minimized)"

# 2. After installation, update system
sudo apt update && sudo apt upgrade -y

# 3. Install required packages
sudo apt install -y \
    dnsmasq \
    tftpd-hpa \
    nginx \
    samba \
    syslinux \
    pxelinux \
    nfs-kernel-server

# 4. Configure static IP
sudo nano /etc/netplan/00-installer-config.yaml
```

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens33:  # Check your interface name with: ip a
      addresses:
        - 192.168.1.1/24
      dhcp4: no
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
```

```bash
sudo netplan apply

# 5. Configure dnsmasq (DHCP + TFTP)
sudo mv /etc/dnsmasq.conf /etc/dnsmasq.conf.backup
sudo nano /etc/dnsmasq.conf
```

```conf
# Disable DNS function
port=0

# Network interface
interface=ens33
bind-interfaces

# DHCP range
dhcp-range=192.168.1.100,192.168.1.200,24h

# Gateway
dhcp-option=3,192.168.1.1

# DNS servers
dhcp-option=6,8.8.8.8,8.8.4.4

# Detect client architecture
dhcp-match=set:bios,option:client-arch,0
dhcp-match=set:efi64,option:client-arch,7
dhcp-match=set:efi64,option:client-arch,9

# Boot files based on architecture
dhcp-boot=tag:bios,pxelinux.0
dhcp-boot=tag:efi64,bootx64.efi

# Enable TFTP
enable-tftp
tftp-root=/srv/tftp

# Logging
log-dhcp
log-queries
```

```bash
# 6. Setup TFTP directory structure
sudo mkdir -p /srv/tftp/{pxelinux.cfg,Boot,sources}

# 7. Copy PXE boot files
sudo cp /usr/lib/PXELINUX/pxelinux.0 /srv/tftp/
sudo cp /usr/lib/syslinux/modules/bios/*.c32 /srv/tftp/

# 8. Create boot menu
sudo nano /srv/tftp/pxelinux.cfg/default
```

```
DEFAULT vesamenu.c32
TIMEOUT 100
PROMPT 0
MENU TITLE === Windows PXE Boot Menu ===
MENU BACKGROUND pxelinux.cfg/background.png

LABEL local
    MENU LABEL ^1) Boot from Local Hard Drive
    LOCALBOOT 0

LABEL windows10
    MENU LABEL ^2) Install Windows 10
    KERNEL memdisk
    INITRD windows10pe.iso
    APPEND iso raw

LABEL windows11
    MENU LABEL ^3) Install Windows 11
    KERNEL memdisk
    INITRD windows11pe.iso
    APPEND iso raw
```

```bash
# 9. Configure Samba for Windows file access
sudo nano /etc/samba/smb.conf
```

```ini
[global]
   workgroup = WORKGROUP
   server string = PXE Server
   security = user
   map to guest = Bad User

[WinInstall]
   path = /srv/windows
   browseable = yes
   read only = yes
   guest ok = yes
   guest only = yes
   create mask = 0644
   directory mask = 0755
```

```bash
# 10. Create Windows files directory
sudo mkdir -p /srv/windows/{win10,win11}
sudo chmod -R 755 /srv/windows

# 11. Restart all services
sudo systemctl restart dnsmasq
sudo systemctl restart tftpd-hpa
sudo systemctl restart smbd
sudo systemctl restart nmbd

# 12. Enable services on boot
sudo systemctl enable dnsmasq
sudo systemctl enable tftpd-hpa
sudo systemctl enable smbd
sudo systemctl enable nmbd

# 13. Disable firewall (or configure it)
sudo ufw disable
# OR allow specific ports:
# sudo ufw allow 67/udp
# sudo ufw allow 69/udp
# sudo ufw allow 445/tcp
# sudo ufw allow 139/tcp

# 14. Check status
sudo systemctl status dnsmasq tftpd-hpa smbd
```
